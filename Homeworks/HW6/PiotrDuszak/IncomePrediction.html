Piotr Duszak - praca domowa 6 - sprawdzanie metryk uczciwości (ang. fairness) na przykładzie predykcji przychodów

W niniejszym notebooku będą testowane modele, na to czy nie dyskryminują jednej z płci na podstawie danych o zarobkach. Na wejściu modelu są takie informacje jak wiek, ukończona edukacja, stan cywilny, zawód, liczba przepracowanych godzin tygodniowo, płeć, a na wyjściu to czy dana osoba zarabia rocznie więcej, czy mniej niż 50 000$

Zostały naczuone 4 modele (3 x modele drzewiaste) oraz sieć neuronowa.

Część właściwa¶

2. Obliczenie metryk uczciwości (fairness)¶

Dla pierwszego modelu (którym jest las losowy) obliczono pięć metryk:

  • statistical parity (SP) - określająca, czy etykieta wyjściowa równa 1 (oznaczająca zarabianie więcej niż 50 000) występuje tyle samo razy dla obydwu płci
  • equal oportunity (EO) - określająca, czy model posiada taki samą czułość (true positive rate) dla obydwu płci
  • predictive parity (PP) - określająca, czy model posiada taki samą samą pozytywną wartość predykcyjną (posotive predictive value) dla obydwu płci
  • predictive equality (PE) - określająca, czy model ma taki sam współczynnik wyników fałszywie dodatnich (false positive rate) dla obydwu płci
  • accuracy equality (AE) - określająca, czy model ma taką sama trafność (accuracy) dla obydwu płci

Jak widać dwie z pięciu metryk (AE oraz PP) mieszczą się w granicach dopuszczalności, ale pozostałe zdecydowanie nie. W szczególości metryka SP jest bardzo niska co oznacza, że szanse mężczyzn na uzyskanie według modelu płacy powyżej 50 000$ rocznie jest znacznie większa niż dla kobiet.

Postanowiono sprawdzić, czy w rzeczywistości istnieje zależność między płcią, a tym ile dana osoba zarabia. I niestety (ale nie zaskakująco) okazało się, że istnieje bardzo duża zależność.

Komentarz:

Niestety pierwszy model okazał się bardzo dyskryminujący kobiety. Na pięć sprawdzonych metryk, tylko dwie mieściły się w dopuszczalnych granicach, a pozostałe trzy były bardzo dalekie od tego. Aczkolwiek nie jest dziwne, że model okazał się dyskryminujący skoro, rzeczywiste dane takie są.

3. Wytrenowanie trzech innych modeli i obliczenie metryk uczciwości¶

Postanowiono skontruować następnujęce modele:

  • sieć neuronową
  • model typu gradient boost
  • zmodyfikowany model lasu losowego w taki sposób, że przed ostateczną predykcją sztucznie zwiększamy szanse kobiet, a zmniejszamy mężczyzn

Komentarz:

Wszystkie modele mają bardzo podobne accuracy (na poziomie 85%). Jak widać nie udało się uzyskać modelu, który spełniałby wszystkie metryki. Trzy na cztery modele posiadają trzy metryki w dopuszczalnych granicach. Żaden model nie posiada dopuszczalnych wartości dla metryki SP oraz PE.

Najlepszy pod tym względem okazał się ręcznie poprawiany losowy las. Po pierwsze metryki EO, PP oraz AE są w dopuszczalnych granicach, a po drugie pozostałe metryki są najbliżej jedynki.

# Funkcje obliczające kolejne metryki uczciwości oraz funkcja agregująca je wszystkie dla danego modelu

def stat_par(y_hat, prot_Female, prot_Male):
  fem_data = (y_hat[prot_Female]==1).mean()
  mal_data = (y_hat[prot_Male]==1).mean()
  return pd.concat([fem_data, mal_data], axis=1).rename(columns={0: 'Female', 1: 'Male'}, index={'income_>50K': 'Statistical parity'})

def equal_opo(y_hat, y, prot_Female, prot_Male):
  y_1 = (y['income_>50K']==1)
  fem_data = (y_hat[prot_Female & y_1]==1).mean()
  mal_data = (y_hat[prot_Male & y_1]==1).mean()
  return pd.concat([fem_data, mal_data], axis=1).rename(columns={0: 'Female', 1: 'Male'}, index={'income_>50K': 'Equal opportunity'})

def pred_par(y_hat, y, prot_Female, prot_Male):
  y_hat_1 = (y_hat['income_>50K']==1)
  fem_data = (y[prot_Female & y_hat_1]==1).mean()
  mal_data = (y[prot_Male & y_hat_1]==1).mean()
  return pd.concat([fem_data, mal_data], axis=1).rename(columns={0: 'Female', 1: 'Male'}, index={'income_>50K': 'Predictive parity'})

def pred_equ(y_hat, y, prot_Female, prot_Male):
  y_0 = (y['income_>50K']==0)
  fem_data = (y_hat[prot_Female & y_0]==1).mean()
  mal_data = (y_hat[prot_Male & y_0]==1).mean()
  return pd.concat([fem_data, mal_data], axis=1).rename(columns={0: 'Female', 1: 'Male'}, index={'income_>50K': 'Predictive equality'})

def acc_equ(y_hat, y, prot_Female, prot_Male):
  fem_data = (y_hat[prot_Female]==y[prot_Female]).mean()
  mal_data = (y_hat[prot_Male]==y[prot_Male]).mean()
  return pd.concat([fem_data, mal_data], axis=1).rename(columns={0: 'Female', 1: 'Male'}, index={'income_>50K': 'Accuracy equality'})

def agreg_all_metrics(y_hat, y, prot_Female, prot_Male):
  fairness_data = []
  fairness_data.append(stat_par(y_hat, data_Female, data_Male))
  fairness_data.append(equal_opo(y_hat, y, data_Female, data_Male))
  fairness_data.append(pred_par(y_hat, y, data_Female, data_Male))
  fairness_data.append(pred_equ(y_hat, y, data_Female, data_Male))
  fairness_data.append(acc_equ(y_hat, y, data_Female, data_Male))
  fairness_data = pd.concat(fairness_data)
  fairness_data['Quotient'] = fairness_data['Female']/fairness_data['Male']
  return fairness_data

  • equal oportunity (EO) - określająca, czy model posiada taki samą czułość (true positive rate) dla obydwu płci
  • predictive parity (PP) - określająca, czy model posiada taki samą samą pozytywną wartość predykcyjną (posotive predictive value) dla obydwu płci
  • predictive equality (PE) - określająca, czy model ma taki sam współczynnik wyników fałszywie dodatnich (false positive rate) dla obydwu płci

def rfc_modif_pred(rfc, X):
